STOCK_INFO Project Overview

I am building a side project called STOCK_INFO.

The project is a US stock analysis and screening platform focused on fundamental data, not short-term trading execution. The goal is to help retail investors quickly understand, compare, and filter stocks using structured financial metrics and strategy-based screening logic.

This project is intended to become a fundamental-oriented stock screener + investment decision support tool with a user-friendly interface for quickly judging whether a stock looks attractive.

1. Core Product Goal

The main purpose of this project is:

to analyze US stocks using fundamental data
to provide a stock screener centered on valuation, profitability, growth, and dividend metrics
to help individual investors make faster and more informed decisions
to build a UI that allows users to understand “what kind of stock this is” at a glance
Current direction

This project is not currently focused on real-time trading execution or day trading UI.

Instead, the product direction is:

real-time trading UI: on hold
fundamental-based filtering and recommendation system: main focus
fast, intuitive UI for retail investors: main focus

So the product should be understood as:

a stock analysis interface that helps users quickly evaluate stocks, rather than a trading terminal

2. High-Level Architecture

The repository is organized like this:

stock_info/
 ├─ next-app      // backend + API server
 ├─ react-app     // frontend UI
 ├─ shared-types  // shared type definitions
Architectural characteristics
frontend and backend are separated
the backend is built with Next.js API routes
the frontend is a separate React + TypeScript application
type definitions are centralized in shared-types
data is delivered primarily through REST APIs
WebSocket-based real-time functionality existed before, but is currently removed or paused

The architecture is intentionally simple and pragmatic. It is designed to support:

stock list fetching
batch or per-symbol fundamentals fetching
frontend rendering and filtering
later expansion into ranking, recommendation, and cached data delivery
3. Data Flow

The current flow is roughly as follows.

1) Stock list API

The frontend first retrieves a list of candidate stocks from:

/api/stocks/hot-tickers

This endpoint is designed to return a list of “hot” stocks, currently based on something like:

top 100 stocks by trading volume
sourced from the EODHD Screener API

This “hot tickers” list acts as the initial universe shown to users.

2) Fundamentals API

For each stock symbol, the frontend requests fundamental data from:

/api/stocks/fundamentals?symbol=AAPL

This endpoint:

calls the EODHD fundamental API
extracts and transforms useful data fields
returns cleaned metrics to the client

The backend does not simply proxy raw API results. The intention is to normalize and expose only the fields the frontend actually needs.

3) Frontend merge and rendering flow

On the frontend side, the main UI flow is roughly:

StockTable.tsx
  ↓
fetchFundamentalsFor()
  ↓
merge symbol list + fundamentals
  ↓
render table

So the client:

fetches a stock universe
fetches the fundamentals for those symbols
merges the data into a renderable format
displays everything in a stock table UI

This is the current core interaction model.

5. Current Core Features
1) Fundamental data table

The main feature at the moment is a stock table that displays financial metrics such as:

PER
EPS
PBR
ROE
dividend yield
market cap
volume
and similar metrics

This allows users to compare multiple stocks side by side.

2) Fundamental filter system

A major planned focus is the filtering system.

Examples of intended filters include:

low PER
high ROE
high EPS growth
high dividend yield
value-oriented filters
growth-oriented filters
stability filters

The filtering system is one of the project’s core features and should become a major part of the product experience.

3) Recommended stock groups

The project also aims to present predefined stock groups, such as:

Hot 100 stocks
Momentum / sharp gainers
High dividend stocks
Earnings growth stocks
Undervalued value stocks
52-week high stocks

These groups are meant to give users immediate entry points instead of forcing them to start from a blank screen.

6. Differentiation / Product Ideas

This project is not only about showing raw numbers. One of the important goals is to make the product easier to understand and more investor-friendly than a plain spreadsheet.

Some key differentiation ideas are:

1) One-line stock summary cards

A feature like:

“What is this stock in one sentence?”

The idea is to summarize a stock very quickly using a concise explanation based on its financial profile.

Example concept:

“Profitable large-cap tech stock with moderate valuation and strong margins”
“High-growth but expensive stock with weak dividend appeal”
“Value-oriented dividend stock with stable cash flow”

This would make the product more approachable for retail users.

2) Investment checklist UI

The project may include a checklist or scorecard style view to visualize whether a stock passes certain conditions.

Examples:

profitability
growth
valuation attractiveness
leverage safety
dividend quality

This could be shown as:

stars
pass/fail badges
checklist rows
category score summaries
3) Famous investor style filters

One of the key ideas is to provide predefined screening strategies inspired by well-known investing styles.

Examples include:

Warren Buffett style
Peter Lynch growth style
Greenblatt Magic Formula
Benjamin Graham low-PER or value style
dividend/stability oriented filters
sector-based filters such as technology-heavy screens

This is an important feature because it makes the screener more opinionated and useful than generic metric filtering.

4) Mock portfolio / watchlist tracking

A future feature is a simulated portfolio or watchlist-based profit tracking system.

This would allow users to:

track selected stocks
simulate returns
compare picked groups over time
monitor ideas without placing real trades

This feature is planned, not the current main focus.
